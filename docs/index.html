kioskwm

Compositor Wayland para kiosk e testes aninhados no desktop.

Instalar Como usar Atalhos Dependências Distros Imagem server Desinstalar GitHub

Instalar

Ubuntu / Debian, arquitetura x86_64:

curl -fsSL https://raw.githubusercontent.com/mdf-leon/kioskwm/main/install/install.sh | bash

Versão específica:

curl -fsSL https://raw.githubusercontent.com/mdf-leon/kioskwm/main/install/install.sh | bash -s -- --version v0.1.0

Depois da instalação, o comando kioskwm fica em /usr/local/bin.

Como usar

Modo aninhado (desenvolvimento / teste no KDE, GNOME, etc.)

Roda dentro de uma janela, usando o compositor pai (Plasma, etc.). Ideal para desenvolver no Konsole:

kioskwm --desktop

Por padrão abre um terminal (Konsole, Alacritty, …) conectado ao socket Wayland do kioskwm.

Modo TTY / kiosk (produção)

Compositor fullscreen direto na placa de vídeo (DRM + libseat). Detectado automaticamente quando você está num VT de hardware (/dev/tty1, tty2, …):

# num tty dedicado, por exemplo após login ou autologin
kioskwm
Importante: o kioskwm é o compositor Wayland. Não precisa de Plasma, GNOME nem sessão Wayland pré-existente no modo TTY. Você só precisa de kernel + DRM, libseat e as dependências listadas abaixo.

Dependências

Runtime (já instaladas pelo install.sh)

libseat1 libinput10 libgbm1 libxkbcommon0 libudev1 libdrm2 \
libwacom9 libevdev2 libmtdev1t64 xwayland

XWayland (xwayland) é necessário para apps X11 (Kate, Krita snap, etc.). Apps Wayland nativas funcionam sem X.

Build (compilar do código)

sudo apt install build-essential pkg-config \
  libseat-dev libinput-dev libudev-dev libxkbcommon-dev \
  libgbm-dev libdrm-dev libegl1-mesa-dev libwayland-dev
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
cargo build --release

Extras recomendados em kiosk

  • Terminal: konsole, alacritty, foot, … (spawn automático)
  • Driver GPU: Mesa (mesa-utils) ou driver proprietário conforme hardware
  • Fontes/cursor: tema de cursor (breeze-cursor-theme ou adwaita-icon-theme)
  • systemd: para desligar/reiniciar pelo painel P1

Imagem server / minimal (sem Wayland pré-instalado)

Cloud images e ISOs “server” normalmente vêm sem ambiente gráfico, sem Wayland e sem display manager. Isso é esperado — o kioskwm substitui o compositor inteiro.

O que você precisa instalar

# base gráfica mínima + kioskwm
curl -fsSL https://raw.githubusercontent.com/mdf-leon/kioskwm/main/install/install.sh | bash

# terminal (escolha um)
sudo apt install -y foot   # leve, Wayland-native
# ou: sudo apt install -y konsole alacritty

# driver e utilitários GPU (Intel/AMD; ajuste se NVIDIA)
sudo apt install -y mesa-utils libgl1

O que configurar

  1. XDG_RUNTIME_DIR — obrigatório. Em autologin systemd: 
    # /etc/systemd/system/kioskwm.service (exemplo)
[Service]
Environment=XDG_RUNTIME_DIR=/run/user/1000
User=kiosk
PAMName=login
TTYPath=/dev/tty1
StandardInput=tty
StandardOutput=journal
ExecStart=/usr/local/bin/kioskwm --no-spawn
Restart=on-failure
  2. Autologin num TTY — edite /etc/systemd/system/getty@tty1.service.d/override.conf ou use conta kiosk com shell que executa kioskwm.
  3. Permissões — usuário precisa acessar DRM/libseat (geralmente grupo video, input, seat conforme distro).
  4. Apps — defina WAYLAND_DISPLAY automaticamente pelo kioskwm; apps clientes conectam ao socket que ele cria (log mostra o nome).
  5. SSH headless — use outro TTY (Ctrl+Alt+F2) ou SSH; o kioskwm ocupa o tty1 com DRM.
Não é necessário: Plasma, GNOME, wayland-protocols do desktop, GDM/SDDM, ou imagem “desktop”.
É necessário: GPU com KMS/DRM, libseat, input (libinput), e ideally systemd-logind ou sessão libseat funcional.

Checklist rápido num Ubuntu Server limpo

curl -fsSL https://raw.githubusercontent.com/mdf-leon/kioskwm/main/install/install.sh | bash

Desinstalar

# remove o binário
sudo rm -f /usr/local/bin/kioskwm

# dependências instaladas pelo script (opcional — podem ser usadas por outros apps)
sudo apt remove -y libseat1 libinput10 libgbm1 libxkbcommon0 libudev1 \
  libdrm2 libwacom9 libevdev2 xwayland

Logs e socket runtime:

rm -f /tmp/kioskwm.log
rm -rf /run/user/$(id -u)/wayland-*   # cuidado: remove sockets wayland do usuário
