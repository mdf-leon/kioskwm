kioskwm

Wayland compositor for kiosk deployments and nested desktop testing.

Install Usage Shortcuts Dependencies Distros Server images Uninstall GitHub

Install

Ubuntu / Debian, x86_64:

curl -fsSL https://raw.githubusercontent.com/mdf-leon/kioskwm/main/install/install.sh | bash

Pin a version:

curl -fsSL https://raw.githubusercontent.com/mdf-leon/kioskwm/main/install/install.sh | bash -s -- --version v0.1.0

After install, kioskwm is available at /usr/local/bin/kioskwm.

Usage

Nested mode (development / testing on KDE, GNOME, etc.)

Runs inside a window using the parent compositor (Plasma, etc.). Ideal for hacking in Konsole:

kioskwm --desktop

By default it spawns a terminal (Konsole, Alacritty, …) connected to kioskwm’s Wayland socket.

TTY / kiosk mode (production)

Fullscreen compositor on the GPU (DRM + libseat). Auto-detected on a hardware VT (/dev/tty1, tty2, …):

# on a dedicated tty, e.g. after login or autologin
kioskwm
Important: kioskwm is the Wayland compositor. You do not need Plasma, GNOME, or an existing Wayland session in TTY mode — only kernel + DRM, libseat, and the dependencies below.

In nested mode (--desktop), the parent compositor (KDE/KWin) may steal global shortcuts. kioskwm tries to inhibit that; if it fails, the evdev fallback still catches P0/P1.

Dependencies

Runtime (installed by install.sh)

libseat1 libinput10 libgbm1 libxkbcommon0 libudev1 libdrm2 \
libwacom9 libevdev2 libmtdev1t64 xwayland

XWayland (xwayland) is required for X11 apps (Kate, Krita snap, etc.). Native Wayland apps work without X.

Build (from source)

sudo apt install build-essential pkg-config \
  libseat-dev libinput-dev libudev-dev libxkbcommon-dev \
  libgbm-dev libdrm-dev libegl1-mesa-dev libwayland-dev
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
cargo build --release

Recommended extras for kiosks

  • Terminal: konsole, alacritty, foot, … (auto-spawn)
  • GPU driver: Mesa (mesa-utils) or proprietary driver as needed
  • Cursors/fonts: cursor theme (breeze-cursor-theme or adwaita-icon-theme)
  • systemd: for shutdown/reboot from the P1 panel

Server / minimal images (no Wayland preinstalled)

Cloud images and “server” ISOs usually ship without a desktop, Wayland, or a display manager. That’s expected — kioskwm replaces the whole compositor stack.

What to install

# minimal graphics base + kioskwm
curl -fsSL https://raw.githubusercontent.com/mdf-leon/kioskwm/main/install/install.sh | bash

# terminal (pick one)
sudo apt install -y foot   # lightweight, Wayland-native
# or: sudo apt install -y konsole alacritty

# GPU driver and utils (Intel/AMD; adjust for NVIDIA)
sudo apt install -y mesa-utils libgl1

What to configure

  1. XDG_RUNTIME_DIR — required. Example systemd unit: 
    # /etc/systemd/system/kioskwm.service
[Service]
Environment=XDG_RUNTIME_DIR=/run/user/1000
User=kiosk
PAMName=login
TTYPath=/dev/tty1
StandardInput=tty
StandardOutput=journal
ExecStart=/usr/local/bin/kioskwm --no-spawn
Restart=on-failure
  2. Autologin on a TTY — override getty@tty1 or use a kiosk user whose shell runs kioskwm.
  3. Permissions — user needs DRM/libseat access (often video, input, seat groups).
  4. Apps — kioskwm sets WAYLAND_DISPLAY; clients connect to the socket it creates (name appears in the log).
  5. SSH headless — use another TTY (Ctrl+Alt+F2) or SSH; kioskwm owns tty1 with DRM.
Not required: Plasma, GNOME, GDM/SDDM, or a “desktop” image.
Required: GPU with KMS/DRM, libseat, input (libinput), and ideally working systemd-logind or a libseat session.

Quick checklist on a clean Ubuntu Server

sudo apt update
curl -fsSL https://raw.githubusercontent.com/mdf-leon/kioskwm/main/install/install.sh | bash
sudo apt install -y foot mesa-utils
# autologin + ExecStart=kioskwm on tty1
# reboot and test

Uninstall

# remove the binary
sudo rm -f /usr/local/bin/kioskwm

# optional — deps installed by the script (may be used elsewhere)
sudo apt remove -y libseat1 libinput10 libgbm1 libxkbcommon0 libudev1 \
  libdrm2 libwacom9 libevdev2 xwayland

Logs and runtime sockets:

rm -f /tmp/kioskwm.log
rm -rf /run/user/$(id -u)/wayland-*   # caution: removes user wayland sockets
